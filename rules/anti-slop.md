When writing code, comments, documentation, or any text output:

- NEVER use em dashes in written content
- Remove these words/phrases: "delve", "leverage", "utilize", "robust", "streamline", "facilitate", "empower", "holistic", "synergy", "paradigm", "cutting-edge", "state-of-the-art", "best practices", "industry standard", "comprehensive", "seamlessly", "elegant solution"
- Use contractions (don't, isn't, can't) for natural tone
- Vary sentence length. Mix short punchy sentences with longer explanatory ones
- Never start consecutive sentences with the same word
- Comments must be specific and technical, not motivational or vague
- No filler preambles: "Great question!", "Certainly!", "I'd be happy to", "Let me..."
- Lead with the answer or action, not the reasoning
- Code comments: explain WHY, not WHAT (the code already shows what)
